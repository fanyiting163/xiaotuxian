# 小兔鲜电商网站 (Xiaotuxian) 🐰

> 基于 HTML5 + CSS3 + JavaScript + JSON 的电商前端项目，包含核心页面布局与完整的业务逻辑交互。

## 📖 项目介绍
这是我跟随黑马程序员前端课程完成的实战项目。
项目从零开始构建了 PC 端电商网页，前期完成了静态页面的布局，后期**引入原生 JavaScript 和 JSON 技术**，实现了用户注册登录、表单验证、数据持久化存储等核心业务功能，模拟了真实电商网站的交互流程。

## 🛠️ 技术栈
* **核心语言：** HTML5, CSS3, JavaScript (ES6+)
* **数据处理：** **JSON (数据序列化/反序列化)**, LocalStorage (本地存储)
* **布局技术：** Flex 布局, 语义化标签, 精灵图, Iconfont
* **交互实现：** DOM 操作, 事件委托, 正则表达式, 定时器

## ✨ 功能特性 (Features)

### 1. 数据管理 (Data Management) 🆕
* **JSON 应用**：
    * 使用 `JSON.stringify()` 将复杂的用户对象序列化为字符串，存入浏览器。
    * 使用 `JSON.parse()` 将读取到的字符串还原为对象，以便在页面中渲染用户信息。
* **本地存储**：利用 `LocalStorage` 实现数据的持久化，刷新页面后用户登录状态依然保持。

### 2. 注册模块 (Register)
实现了严谨的表单验证逻辑（使用正则）：
* **短信验证码**：模拟 60s 倒计时交互逻辑。
* **输入验证**：
    * **用户名**：校验 6-10 位字母/数字/下划线。
    * **手机号**：标准 11 位手机号正则校验。
    * **密码强度**：校验 6-20 位字符。
    * **二次确认**：自动检测两次密码输入是否一致。
* **提交守卫**：只有所有表单项验证通过（返回 `true`）且勾选了注册协议，提交按钮才会触发数据写入操作。

### 3. 登录模块 (Login)
* **交互逻辑**：
    * Tab 栏切换登录方式。
    * 阻止表单默认提交，改为 JS 控制逻辑。
* **登录验证**：
    * 校验非空及协议勾选状态。
    * **数据写入**：登录成功后，将用户信息封装为 JSON 格式存入本地，并跳转至首页。

### 4. 首页状态 (Home)
* **状态渲染**：
    * 页面加载时自动读取本地存储的 JSON 数据。
    * **已登录状态**：DOM 操作动态修改头部，显示“用户名”及“退出登录”。
    * **未登录状态**：显示默认的“登录/注册”入口。
* **退出登录**：点击退出后清除本地存储数据，并重置页面结构。
